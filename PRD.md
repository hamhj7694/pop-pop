# PRD: TOKTOK — 수집형 디지털 뽁뽁이 힐링 서비스

## 1. 제품 개요

### 1.1 서비스명

TOKTOK
가칭: 톡!톡!

### 1.2 한 줄 정의

손맛, 사운드, 진동, 이펙트로 뽁뽁이를 터뜨리며 기분 좋은 콘텐츠와 아이템을 수집하는 디지털 힐링 장난감 서비스.

### 1.3 제품 포지션

이 서비스는 단순한 명언 앱이나 힐링 콘텐츠 플랫폼이 아니다.

핵심은 **뽁뽁이를 터뜨리는 감각적 만족**이다.
기분 좋은 문구, 이미지, 아이템, 기프티콘, 협찬 보상 등은 터뜨리는 경험을 강화하는 수집형 보상 요소로 작동한다.

### 1.4 핵심 방향

사용자는 아무 생각 없이 뽁뽁이를 터뜨린다.
터뜨리는 과정에서 사운드, 진동, 파티클, 콤보, 피버타임이 발생한다.
가끔 기분 좋은 콘텐츠나 아이템이 떨어지고, 사용자는 이를 나중에 수집함에서 확인한다.

---

## 2. 제품 목표

### 2.1 Primary Goal

사용자가 짧은 시간 안에 스트레스를 해소하고, 1분 이상 지속적으로 뽁뽁이를 터뜨리게 만든다.

### 2.2 핵심 가치 우선순위

1. 손맛
2. 감각적 만족
3. 랜덤 보상
4. 수집 욕구
5. 기분 좋은 힐링 콘텐츠
6. 커뮤니티 기반 콘텐츠 확장
7. 수익화

### 2.3 성공 지표

* 평균 세션 시간 3분 이상
* 1회 세션 평균 터뜨린 개수 300개 이상
* 보상 획득 경험률 70% 이상
* 도감 재방문율 20% 이상
* 7일 재방문율 20% 이상
* 사운드/진동 ON 유지율 50% 이상

---

## 3. 핵심 사용자 경험

### 3.1 기본 경험

사용자가 서비스에 접속하면 화면 전체에 뽁뽁이가 배치된다.

사용자는 클릭, 터치, 드래그로 뽁뽁이를 터뜨린다.

뽁뽁이를 터뜨릴 때마다 다음 반응이 발생한다.

* 버블 눌림
* 터짐 애니메이션
* 사운드
* 모바일 진동
* 파티클
* 콤보 증가
* 랜덤 보상 드랍 가능성

### 3.2 중요한 UX 원칙

보상은 즉시 팝업으로 뜨지 않는다.

뽁뽁이를 터뜨리는 흐름을 방해하면 안 된다.

보상은 터진 자리에서 떨어지거나 화면 하단에 쌓인다.

사용자는 원할 때만 “오늘 주운 것들”을 열어 확인한다.

## 4. 핵심 기능 요구사항

## F001 뽁뽁이 생성

### 설명

화면 전체에 뽁뽁이 버블을 자동 배치한다.

### 요구사항

* 반응형 배치
* 모바일 우선 설계
* 화면 크기에 따라 버블 수 자동 조정
* 터진 버블과 남은 버블 상태 관리
* 일정 개수 이하로 줄어들면 새 판 생성 가능

---

## F002 뽁뽁이 터뜨리기

### 설명

사용자가 버블을 누르면 해당 버블이 터진다.

### 동작

1. 사용자가 버블 클릭/터치
2. 버블 눌림 애니메이션
3. 터짐 애니메이션
4. 사운드 재생
5. 모바일이면 진동 발생
6. 파티클 발생
7. 버블 상태를 popped로 변경
8. 콤보 카운트 증가
9. 랜덤 보상 여부 계산

---

## F003 연속 터뜨리기

### 설명

사용자가 드래그하거나 손가락으로 쓸면 여러 버블이 연속으로 터진다.

### 요구사항

* 마우스 드래그 지원
* 모바일 터치 드래그 지원
* 이미 터진 버블은 중복 처리하지 않음
* 빠르게 터뜨려도 프레임 저하 최소화
* 연속 터뜨리기 중 사운드가 과도하게 겹치지 않도록 제어

---

## F004 사운드 시스템

### 설명

뽁뽩이 터질 때 짧고 기분 좋은 사운드를 재생한다.

### 요구사항

* 기본 사운드 ON
* 설정에서 ON/OFF 가능
* 사운드 볼륨 조절 가능
* 여러 종류의 터짐 사운드 랜덤 재생
* 피버타임 시 특수 사운드 적용 가능

### 기본 사운드 예시

* 톡
* 뽁
* 팡
* 뿅

---

## F005 모바일 진동 시스템

### 설명

모바일 기기에서 버블이 터질 때 햅틱 피드백을 제공한다.

### 요구사항

* 기본 진동 ON
* 설정에서 ON/OFF 가능
* 브라우저가 Vibration API를 지원할 때만 작동
* 미지원 기기에서는 조용히 비활성화
* 사운드 설정과 별개로 독립 제어

### 진동 패턴

일반 버블:

* 10~15ms

희귀 보상 드랍:

* 30ms

초희귀 보상 드랍:

* 40ms + 짧은 간격 + 20ms

피버타임 시작:

* 짧은 패턴 진동

## F010 콤보 시스템

### 설명

일정 시간 안에 계속 뽁뽁이를 터뜨리면 콤보가 유지된다.

### 목적

연속 터뜨리기의 재미를 강화한다.

### 원칙

콤보는 경쟁 요소가 아니라 축하 요소다.
실패해도 손해가 없어야 한다.
힐링 서비스이므로 과한 스트레스나 페널티를 만들지 않는다.

### 조건

* 버블을 터뜨릴 때 콤보 +1
* 일정 시간 입력이 없으면 콤보 종료
* 콤보 종료 시 결과 표시
* 실패 페널티 없음

### 콤보 단계

* 10 Combo
* 50 Combo
* 100 Combo
* 300 Combo
* 500 Combo
* 1000 Combo

### 콤보 보상

10 Combo:

* 일반 보상 드랍

50 Combo:

* 일반 보상 2개 드랍 가능

100 Combo:

* 희귀 확률 증가

300 Combo:

* 희귀 상자 드랍 가능

500 Combo:

* 희귀 보상 확정 드랍

1000 Combo:

* 특별 보상 또는 한정 아이템 드랍

---

## F011 콤보 종료 화면

### 설명

콤보가 끊기면 가볍게 성과를 보여준다.

### 예시

523 Combo 달성!

오늘 가장 긴 콤보예요.

획득한 보상:

* 별조각 12개
* 좋은 문구 3개
* 희귀 카드 1개

### 주의사항

* 광고 부활 기능은 MVP에서 제외
* 콤보 실패를 부정적으로 표현하지 않음
* “실패”보다 “기록 달성”으로 표현

## F012 피버타임

### 설명

뽁뽁이를 터뜨리다 보면 낮은 확률로 자동 발동되는 특별 모드.

### 목적

예측 불가능한 즐거움과 순간적인 몰입감을 만든다.

### 발동 방식

* 일반 터뜨리기 중 낮은 확률로 자동 발동
* 콤보 보상으로 발동 가능
* 특정 아이템으로 발동 가능

### 기본 확률 예시

* 일반 터뜨리기마다 0.3~0.5%
* 콤보 300 이상일 때 발동 확률 소폭 증가

### 지속 시간

* 기본 10초
* 추후 아이템 또는 프리미엄 효과로 확장 가능

### 피버타임 효과

* 보상 등장 확률 증가
* 희귀 보상 확률 증가
* 터짐 이펙트 강화
* 배경 연출 변화
* 사운드 변화
* 진동 패턴 강화
* 피버 전용 보상 등장 가능

### 확률 예시

일반 보상 등장률:

* 기존 대비 3배

희귀 보상 등장률:

* 기존 대비 2배

초희귀 보상 등장률:

* 기존 대비 1.2배

## F019 사용자 콘텐츠 제보

### 설명

사용자가 “기분 좋은 것”을 제보할 수 있다.

### 제보 가능 콘텐츠

* 짧은 문구
* 사진
* 그림
* GIF
* 짧은 음성
* 좋은 순간
* 귀여운 이미지
* 응원 메시지

### 핵심 원칙

사용자 제출 콘텐츠는 바로 공개하지 않는다.
관리자 검수 후 승인된 콘텐츠만 보상 풀에 등록된다.

### 이유

## F023 광고 정책

### MVP 기준

광고는 넣지 않는다.

### 이유

서비스의 핵심 감정선은 기분 좋음, 몰입, 손맛이다.
광고는 이 흐름을 깨뜨릴 가능성이 높다.

### 추후 가능 방식

광고를 넣는다면 다음 방식만 고려한다.

* 선택형 광고
* 보상 2배 광고
* 무료 사용자를 위한 선택형 후원 광고
* 브랜드 협찬 아이템
* 이벤트 응모권

### 금지 방식

* 강제 전면 광고
* 팝업 광고
* 터뜨리는 중간 광고
* 보상인 척하는 광고
* 콤보 부활 광고

## 13. 최종 제품 정의

TOKTOK은 힐링 문구를 읽는 서비스가 아니다.

TOKTOK은 온라인에서 뽁뽁이를 터뜨리는 손맛을 중심으로, 사운드, 진동, 파티클, 콤보, 피버타임, 수집 보상을 결합한 디지털 힐링 장난감이다.

제품의 성패는 콘텐츠 양보다 터뜨리는 감각의 완성도에 달려 있다.

따라서 MVP에서 가장 중요한 것은 다음 네 가지다.

1. 터뜨리는 손맛
2. 연속 터뜨리기의 쾌감
3. 기분 좋은 감각 효과
4. 수집 보상의 기대감

최종 방향은 다음과 같다.

**기분 좋은 것들을 수집하는 수집형 디지털 뽁뽁이 서비스.**
