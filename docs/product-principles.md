# 제품 원칙

## 제품 정의

TOKTOK은 온라인에서 뽁뽁이를 터뜨리는 손맛을 중심으로 사운드, 진동, 파티클, 콤보, 피버타임, 수집 보상을 결합한 디지털 힐링 장난감이다.

힐링 문구를 읽는 서비스가 아니라, 기분 좋은 감각 경험을 반복하게 만드는 서비스다.

## 의사결정 우선순위

개발 중 선택지가 충돌하면 아래 순서를 따른다.

1. 손맛
2. 감각적 만족
3. 랜덤 보상
4. 수집 욕구
5. 기분 좋은 힐링 콘텐츠
6. 커뮤니티 기반 콘텐츠 확장
7. 수익화

## MVP 목표

MVP의 성공은 콘텐츠 양보다 터뜨리는 감각의 완성도에 달려 있다.

반드시 먼저 완성해야 하는 네 가지는 다음과 같다.

1. 터뜨리는 손맛
2. 연속 터뜨리기의 쾌감
3. 기분 좋은 감각 효과
4. 수집 보상의 기대감

## 개발 단계

1. 핵심 손맛: 버블 배치, 클릭/터치, 드래그 연속 터뜨리기, 기본 사운드, 기본 진동, 기본 파티클
2. 보상 루프: 랜덤 보상 드랍, 보상 낙하, 오늘 주운 것들, 도감 저장
3. 몰입 시스템: 콤보, 콤보 보상, 피버타임, 피버 전용 보상
4. 운영 기능: 관리자 콘텐츠 등록, 사용자 제보, 승인/거절, 보상 풀 관리
5. 수익화 확장: 프리미엄 테마, 효과, 사운드팩, 협찬 보상

## 반드시 지킬 경험

- 사용자는 접속 직후 별도 학습 없이 바로 터뜨릴 수 있어야 한다.
- 보상은 즉시 팝업으로 뜨지 않는다.
- 보상은 터진 자리에서 떨어지거나 화면 하단에 쌓여야 한다.
- 사용자는 원할 때만 `오늘 주운 것들`을 열어 확인한다.
- 콤보는 경쟁이나 실패 처벌이 아니라 축하 요소로 표현한다.
- 실패, 손해, 압박, 놓침을 강조하지 않는다.

## MVP 제외 항목

아래 기능은 PRD상 MVP 제외 항목이므로 별도 요청이 없으면 구현하지 않는다.

- 실시간 채팅
- 친구 기능
- 경쟁 랭킹
- 강제 광고
- 결제
- 복잡한 커뮤니티
- 앱 출시
- 푸시 알림
- AI 추천

## 광고와 수익화 기준

MVP에는 광고를 넣지 않는다.

향후 광고를 검토하더라도 선택형 광고, 보상 2배 광고, 선택형 후원 광고, 브랜드 협찬 아이템, 이벤트 응모권만 고려한다.

금지 방식은 다음과 같다.

- 강제 전면 광고
- 팝업 광고
- 터뜨리는 중간 광고
- 보상인 척하는 광고
- 콤보 부활 광고
