# 인터랙션 규칙

## 핵심 목적

모든 인터랙션은 짧고 반복 가능한 만족감을 만들어야 한다. 클릭, 터치, 드래그가 모두 즉각적으로 반응해야 하며, 모바일 터치 드래그를 최우선 시나리오로 본다.

## 버블 생성

- 화면 전체에 버블을 자동 배치한다.
- 반응형 배치를 기본으로 한다.
- 모바일 우선으로 버블 크기, 간격, 터치 타깃을 잡는다.
- 화면 크기에 따라 버블 수를 자동 조정한다.
- 터진 버블과 남은 버블 상태를 명확히 분리한다.
- 터진 버블은 짧은 시간 뒤 같은 자리에서 다시 터뜨릴 수 있게 재생성한다.

## 터뜨리기 입력

버블이 터지는 기본 흐름은 다음 순서를 따른다.

1. 사용자 클릭 또는 터치 입력
2. 버블 눌림 애니메이션
3. 터짐 애니메이션
4. 사운드 재생
5. 모바일 진동 발생
6. 파티클 발생
7. 버블 상태를 `popped`로 변경
8. 콤보 카운트 증가
9. 랜덤 보상 여부 계산

## 연속 터뜨리기

- 마우스 드래그와 모바일 터치 드래그를 모두 지원한다.
- 이미 터진 버블은 다시 처리하지 않는다.
- 빠른 입력 중에도 프레임 저하를 최소화한다.
- 연속 입력 중 사운드가 과도하게 겹치지 않도록 재생 간격, 볼륨, 동시 재생 수를 제어한다.
- 드래그 중 터뜨림 판정은 사용자가 실제로 버블을 쓸고 있다는 느낌을 줘야 한다.

## 사운드

- 기본값은 ON이다.
- 설정에서 ON/OFF와 볼륨 조절이 가능해야 한다.
- 터짐 사운드는 짧고 기분 좋아야 한다.
- 여러 종류의 사운드를 랜덤 재생할 수 있다.
- 피버타임에는 특수 사운드를 적용할 수 있다.
- 사운드를 꺼도 게임 진행과 보상 경험이 훼손되면 안 된다.

## 진동

- 기본값은 ON이다.
- 사운드와 독립적으로 ON/OFF 가능해야 한다.
- 브라우저가 Vibration API를 지원할 때만 작동한다.
- 미지원 기기에서는 오류나 안내 팝업 없이 조용히 비활성화한다.
- 일반 버블은 10~15ms 수준의 짧은 진동을 사용한다.
- 희귀 보상은 30ms, 초희귀 보상은 `40ms + 짧은 간격 + 20ms` 패턴을 사용할 수 있다.
- 피버타임 시작은 짧은 패턴 진동으로 구분한다.

## 애니메이션과 파티클

- 터짐 반응은 즉시 시작되어야 한다.
- 파티클은 기분 좋은 보조 효과이며 입력 판정을 늦추면 안 된다.
- 과도한 파티클로 화면을 가리거나 성능을 떨어뜨리지 않는다.
- 희귀도, 콤보, 피버타임에 따라 이펙트 강도를 차등 적용할 수 있다.
- 사용자가 이펙트 강도를 조절할 수 있어야 한다.

## 설정

설정은 감각 요소를 직접 제어하기 위한 기능이다.

필수 항목은 다음과 같다.

- 사운드 ON/OFF
- 사운드 볼륨
- 진동 ON/OFF
- 이펙트 강도
- 테마 선택
- 효과팩 선택
- 사운드팩 선택
