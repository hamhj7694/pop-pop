# 품질 규칙

## 성능 목표

- 초기 로딩은 2초 이하를 목표로 한다.
- 주요 인터랙션은 60FPS에 가깝게 유지한다.
- 동시에 많은 버블이 터져도 프레임 저하를 최소화한다.
- 사운드 중복 재생을 제한한다.
- 파티클 과다 생성을 방지한다.
- 입력 처리와 애니메이션은 사용자가 지연을 느끼지 않도록 분리한다.

## 성능 체크 기준

다음 작업은 성능 영향을 반드시 확인한다.

- 버블 배치 알고리즘 변경
- 드래그 연속 터뜨리기 변경
- 파티클 수 또는 애니메이션 변경
- 사운드 재생 방식 변경
- Zustand store 구조 변경
- 피버타임 연출 변경

## 접근성

- 사운드 없이도 플레이 가능해야 한다.
- 진동 없이도 플레이 가능해야 한다.
- 시각 효과 강도를 조절할 수 있어야 한다.
- 색상만으로 희귀도나 상태를 구분하지 않는다.
- 설정, 도감, 오늘 주운 것들은 키보드와 스크린 리더 접근성을 고려한다.
- 움직임에 민감한 사용자를 위해 이펙트 강도를 낮추는 옵션을 유지한다.
- 콤보, 보상 발견, 문구 토스트가 동시에 표시되어도 서로 겹치지 않아야 한다.
- 비필수 알림 레이어는 포인터 입력을 가로채지 않아야 한다.

## 모바일 품질

- 모바일 터치 드래그를 우선 테스트한다.
- 한 손 조작이 가능해야 한다.
- 작은 화면에서도 오늘 주운 것들, 설정, 도감 접근이 가능해야 한다.
- 진동 미지원 기기에서 오류 없이 동작해야 한다.
- 화면 회전 또는 크기 변경 시 버블 배치가 깨지지 않아야 한다.

## 운영 안정성

- 사용자 제보 콘텐츠는 검수 전 공개하지 않는다.
- 저작권 위반 가능성이 있는 콘텐츠는 승인하지 않는다.
- 부적절한 콘텐츠 차단 경로를 둔다.
- 협찬/경품 보상은 운영자가 비활성화할 수 있어야 한다.
- 보상 풀 변경이 기존 획득 기록을 깨뜨리지 않아야 한다.

## 테스트 기준

구현 범위에 따라 다음을 확인한다.

- 버블 클릭/터치로 정확히 한 번만 터지는지
- 드래그로 여러 버블이 자연스럽게 터지는지
- 이미 터진 버블이 중복 처리되지 않는지
- 사운드 OFF, 진동 OFF 상태가 지켜지는지
- 보상이 팝업이 아니라 드랍/쌓임 방식으로 보이는지
- 오늘 주운 것들 카운트가 획득 수와 일치하는지
- 콤보 종료가 부정적 표현 없이 표시되는지
- 콤보 알림과 보상/문구 알림이 동시에 떠도 한쪽이 아래로 밀려 겹치지 않는지
- 피버타임 시작/종료가 명확히 표시되는지
- 희귀도 표시가 색상 외 단서도 제공하는지
- 분석 이벤트가 중복 과다 전송되지 않는지

## 분석 이벤트 품질

분석 이벤트는 제품 개선을 위한 최소 정보만 담는다.

- 개인식별정보를 직접 보내지 않는다.
- 고빈도 이벤트인 `bubble_popped`는 샘플링, 배치, 집계 전송을 고려한다.
- 이벤트 이름은 `docs/frontend-architecture-rules.md`의 목록을 우선한다.
- 콤보, 피버타임, 보상 획득 이벤트는 제품 지표와 연결되도록 payload를 설계한다.
